These packages provide Java-based tools to manipulate and transform DAVE-ML source files.

Overview

There are presently three subpackages in this distribution:

  • The gov.nasa.daveml.DAVE package provides a basic utility to parse and evaluate Dynamic Aerospace Vehicle Exchange Markup Language (DAVE-ML) files. The package provides some features as a stand-alone program, but can be extended by other packages to add additional capabilities.

  • The gov.nasa.daveml.DAVE2SL package builds on parsing capabilities provided in the gov.nasa.daveml.DAVE package to provide a utility to convert Dynamic Aerospace Vehicle Exchange Markup Language (DAVE-ML) files into MATLAB® Simulink® .mdl diagrams.

  • The gov.nasa.daveml.sweeper package builds on the parsing capabilities provided by the gov.nasa.daveml.DAVE package to provide a visualization tool for a complete model; it shows 3D surface plots of any model output as a function of any two inputs.

The DAVE2SL package requires the gov.nasa.daveml.DAVE package, which in turn depends on the org.jdom package, a product of the JDOM project, a Java-based XML document parser. See {@link http://www.jdom.org} for more information about JDOM.

The ant tool produces a single DAVEtools.jar that includes DAVE2SL (default entry point), DAVE, ModelSweeper and the JDom and Xerces .jar contents.

See {@link http://daveml.org} for more information about the DAVE-ML standard.

The latest version of DAVEtools can be downloaded from {@link GitHub}.

Revision history

This is DAVEtools version 0.9.8

Changes since version 0.9.7

  • Updated headers to change copyright, new email
  • Cleaned up some Java syntax.
  • Switched from junit3 to 4.
  • Improved error messages in MatFileWriter and elsewhere.
  • Added unit test of switch logic block.
  • Updated HowToRelease notes.
  • "evaluate" command output now includes internal varID
  • Added message with count of cycles required to sort blocks into execution order.

Changes since version 0.9.6

  • Added support for Logic blocks
  • Added ignore check-case option (-x or --no-checkcase) to both DAVE and DAVE2SL so processing will continue even if check-cases are not passing; useful for debugging.
  • Reduced inter-block spacing for Simulink(r) blocks to allow for larger models in a single diagram.

Changes since version 0.9.5

  • Changed to use newer PreLookup and Interpolation_n-D blocks
  • Corrected most 'if', 'while' statements in source to use braces.
  • Changed exception handling to deal with syntax errors at a lower level.
  • Changed order of import statements to make NetBeans happy
  • Deprecated Block.getDests() and Block.getSource() in favor of getDestBlocks() and getSourceBlocks() to remove ambiguity of Signal vs. Block being retrieved.
  • Maybe fixed vertical offset of top-level subsystem?

Changes since version 0.9.4

  • Changed behavior of -i flag to DAVE: in addition to creating an XML snippet with internal values for any checkcase successfully passed, it will also spit out useful intermediate values if the -e (evaluate) flag is also used.
  • Corrected bug in creation command for N-d tables larger than 4 (missing comma).

Changes since version 0.9.3

  • Corrected bug in creation command for N-d tables larger than 4 (missing comma).
  • Added new utilities: DAVE2POST, to generate POST-II input decks and code sections; and DAVE2OTIS, to generate OTIS4 input decks. Added FORTRAN and C code snippet writers, to support these.
  • Added selectionFlag to Block and selectOutput(), getSelectedBlocks() to Model.
  • Tweaked tests to run under Windows XP.
  • Upgraded for use with NetBeans 7.1.1 and Java SE 1.6.
  • Fixed incompatibility with Matlab/Simulink 2011b.
  • Bug fix for correct implementation of inequality operator block in Simulink

Changes since version 0.9.2

  • Added support for MathML2's <min>, <max>, <ceiling> and <floor> elements. Added count of checkcases to dave's statistics summary (when used with the -c option. Added support for UML diagram generation for DAVEtools development. Separated the CHANGELOG from the README files; the README files were rewritten to use asciidoc.

Changes since version 0.9.1

  • Fixed bug: DAVE handles binary <minus> (as well as unary <minus>) apply elements in <variableDef> calculation elements.
  • Fixed bug: DAVE2SL saves function table data using the gtID of the table (mandatory attribute as of DAVE-ML 2.0RC4) instead of the optional name attribute. Previously, unnamed tables would share the same data array in the setup script, with bad results.
  • Fixed bug: exit codes are now 1 for error and 0 for no error; these used to be non-standard -1 for no error and 0 for error. Confused with VAX/VMS return codes, I think!
  • Fixed bug: ModelSweeper now doesn't bomb when looking at a model that has no defined checkcases.

Changes since version 0.9.0

  • Support minValue and maxValue attributes on variableDef (DAVE-ML 2.0RC4)

Changes since version 0.8.1

  • Support full use of parsing <piecewise> elements with multiple <piece>s, per MathML DTD. Previously DAVE was limited to a single <piece>/<otherwise> pair.
  • Changed to support default DAVE-ML namespace attribute
  • Moved to Netbeans development environment
  • Added entity definitions to template types ArrayList and Iterator
  • Now including JDOM and Xerces library in DAVEtools.jar file
  • Renamed DAVE2SL.jar to DAVEtools.jar; includes DAVE, jdom, and xerces jar-file contents in a single Jar file for a standalone DAVE to Simulink conversion tool
  • Added ModelSweeper utility program

Changes since version 0.8.0

  • Added support for logicals (not, and, or, xor) per MathML DTD
  • Switched to 'ant' from 'make'

Changes since version 0.7

  • Added capability to extend MathML with known DAVE-ML operator atan2
  • Recognize arguments with either -c or --count (was just -c or -count)
  • Supports immediate <ci> and <cn> subelements in a <math> element, per MathML
  • Failing verification checkcases report miscomparing values
  • Now available under NASA open-source license.

Changes since verision 0.6

  • DAVE2SL now generates a _create m-script instead of .mdl file
  • Data set is now stored in model workspace instead of base workspace
  • Added both initial unit tests and application test cases

Changes since verison 0.5

  • General cleanup of documentation
